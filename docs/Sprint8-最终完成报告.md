## 🔄 完整消息流程

### 用户发送消息流程

```
1. 用户输入消息
        ↓
2. handleSend() 创建用户消息
        ↓
3. 添加到 messages 数组
        ↓
4. 创建AI消息（loading状态）
        ↓
5. sendMessage() 通过WebSocket发送
        ↓
6. 后端接收并验证Token
        ↓
7. 调用AIStreamingService
        ↓
8. 调用通义千问API
        ↓
9. 流式返回响应

前端接收流程：
        ↓
10. onStreamStart 事件
        ↓
11. onStreamChunk 事件（多次）
        ↓
12. currentResponse 累积内容
        ↓
13. 实时渲染到界面
        ↓
14. onStreamComplete 事件
        ↓
15. 更新消息内容
        ↓
16. 保存到localStorage
```

## 💾 数据存储方案

### localStorage存储

**存储项**:
```javascript
// 1. 当前对话
'currentChat': JSON.stringify(messages)

// 2. 历史记录（最多50条）
'chatHistory': JSON.stringify([
  {
    id: 'conv_xxx',
    title: '对话标题',
    preview: '预览',
    messages: [...],
    timestamp: xxx
  }
])

// 3. 用户设置
'aiChatSettings': JSON.stringify({
  keepContext: true,
  autoSave: true
})
```

**存储时机**:
- 实时：消息变化时
- 手动：清空对话时
- 自动：组件卸载时

**数据大小估算**:
- 单条消息: ~500 bytes
- 单个对话(50条): ~25KB
- 50个历史记录: ~1.25MB
- 总计: <2MB (localStorage限制5MB)
